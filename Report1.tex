\section* {Постановка задачи}
\parindent=1cm

\hspace{\parindent} Требуется разработать программный пакет для редактирования шрифтов и просмотра результатов с возможностью тестовой печати текста созданным шрифтом. Пользователь должен иметь возможность:
\begin{itemize}
  \item Создать новый шрифт
  \item Редактировать внешний вид букв шрифта (буквы задаются набором кривых Безье) 
  \item Сохранить внесённые изменения с возможностью последующей загрузки
  \item Опробовать шрифт во встроенном текстовом редакторе. 
\end{itemize}

\section* {Общая архитектура приложения}
\hspace{\parindent} Программа разработана по прицнипу Model-View-Controller. Глобально разделена на две подпрограммы - подпрограмму редактирвоания шрифта и подпрограмму тестирования созданных шрифтов.

\section* {Используемая модель}
\hspace{\parindent}Для решения поставленной задачи была применена теория кривых Безье. Напомним, что кривая Безье m-го порядка задаётся параметрическим уравнением
$$R(t)=\sum\limits_{i=0}^{m}{m \choose i} t^i(1-t)^{m-i}P_i, t \in [0,1]$$
где $P_i \in \mathbb{R}$ - базисные точки кривой.
Важным при использовании кривых является понятие геометрической непрерывности. \\
\textbf{def } Кривая называется $G^0$-непрерывной, если она непрерывна. \\
\textbf{def } Кривая называется $G^1$-непрерывной, если она непрерывна, если её касательная меняется непрерывно при изменении параметра. \\
\textbf{def } Кривая называется $G^2$-непрерывной, если она $G^1$-непрерывна и её вектор кривизны меняется непрерывно. \\

Заметим, что при разработке шрфитов наиболее интересны геометрически непрерывные кривые разных порядков, ибо они позволяют достичь необходимоых виузальный свойств кривой. В частности, границы букв должны состоять из $G^0$-кривых. Если требуется гладкость границ, то стоит использовать $G^1$-кривые. Очевидно также, что на плоскости любая \\$G^1$- кривая будет $G^2$-кривой

В нашей работе мы использовали кривые Безье третьего порядка, задаваемые четырьмя базисными точками на плоскости. Нетрудно понять, что любая кривая Безье априори является $G^1$-непрерывной. Однако, чтобы таким свойством обладала кривая, составленная из кривых Безье, требуется выполнение определённых граничных условий. 

Для $G^1$-непрерывности составной кривой требуется, чтобы в точке сочленения направляющие векторы касательных были коллинеарны, что возможно только при условии нахождения трех последовательных крайних точек (точки сочленения и её соседей) на одной прямой.  
 
Для хранения модели были реализованы два класса. Класс \tt{Curve} \normalfont преставляет собой простую модель кривой Безье, а класс  Модели-Представления  \tt{DrawableCurve}\normalfont\space- его обёртку, предоставляющую необходимые интерфейсы для отображения кривой на экране. Составная кривая хранится как объект, содержащий список объектов \tt{DrawableCurve}\normalfont\space, а также вспомогательную информацию о составной кривой (например, замкнута ли кривая).
Класс\space\tt{Letter}\normalfont\space представляет модель буквы шрифта. Класс \tt Font\normalfont, хранящий шрифты, созданные пользователем, предоставляющий интерфейсы загрузки и сохранения в файл, отображения в текстовом редакторе с возможностью настройки размера каждой отдельной буквы.
 

\section* {Контроллер}
\hspace{\parindent} Реализованный класс контроллера составляет ядро бизнес-логики. Он отвечает за управление кривыми на экране исходя из передаваемых ему запросов от классов представления. Решализуемая им функциональность:
\begin{itemize}
  \item Добавление кривой Безье
  \item Перетаскивание кривой Безье по экрану
  \item Соединение кривых Безье в одну кривую
  \item Логгирование действий пользователя для возможного отката действий
  \item Очистка экрана
  \item Отмена последнего действия на холсте
\end{itemize}

\section* {Управление}
\subsection*{Редкатор шрифта}
\hspace{\parindent} В подпрограмме редактирвоания шрифта выбор действия осуществляется переключателем Move/Draw. Режим Move позволяет двигать нарисованные кривые, Draw - рисовать новые.

Рисование кривых осуществляется нажатием левой кнопки мыши с дальнейшим движением с зажатой кнопкой указателя мыши по окну программы. Чтобы нарисовать несколько соединённых кривых, достаточно начинать каждую следующий кривую в граничной базисной точке одной из нарисованных линий. Заметим, что граничная базисная точка не должна быть уже соединена с какой-либо другой точкой, чтобы кривые зацепились.

Движение кривых можно производить либо зацепив любую базисную точку (внутренние базисные точки отображаются мелкими серыми, а граничные - крупными чёрными), либо зацепив указателем чёрную линию кривой. В первом случае будет передвинута только зацепленная точка (в случае, режима $G^1$-непрерывности будут перемещаться и соседние точки), во втором - вся кривая как единое целое. 

Соединять кривые можно, сблизив их граничные базисные точки, к которым ещё не присоединеные никакие другие кривые. Сблизить граничные точки можно передвигая кривые. 

Также имеется возможность подразбить нарисованную кривую на две части, быстро щелкнув дважды левой кнопкой мыши по чёрной линии кривой.

Отменить последнее действие можно, нажав кнопку <<Undo>>. Действия можно отменять несколько раз подряд.

После того, как редактирвоание буквы завершено, её можно сохранить, нажав кнопку <<Save Letter>>. Предварительно надо ввести, что за буква нарисована, в белом окне ввода <<Letter>> слева. 

Выбор гладкости осуществляется флажком <<G1 smooth>>. 

\section* {Дополнительные исследования}
Для анализа того, сколько операций совершает тот или иной метод, предлагается добавить счетчики операций (скорее всего, арифметических), дабы понять истинную амортизационную  оценку сложности. Далее, думаю можно сделать простенький генератор случайных матриц ограничений и параметров линейной задачи для того, чтобы получить большую статистическую выборку этих сложностей, и взяв например среднее выборочное, дать устойчивую оценку сложностей алгоритмов.
